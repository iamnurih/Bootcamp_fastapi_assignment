
## Fields 란 무엇인가? 
- Pydantic 모델의 각 속성에 추가적인 설정과 제약 조건을 부여하는 함수. 
- 데이터 유효성 검사 규칙뿐만 아니라 API 문서에 표시될 설명이나 예시 값 등도 추가 가능. 
- 타입 검사를 넘어서 구체적인 데이터 유효성 검사 규칙을 추가하고 싶을 때 사용 


```
from pydantic import BaseModel, Field

class Item(BaseModel): # 👇 Field를 여기에 사용 
	name: str = Field(..., min_length=3)


``` 


- **... (Ellipsis)의 의미** `Field`를 사용하면서 이 필드가 **필수(Required)라는 것을 명시**하고 싶을 때 `...`를 첫 번째 인자로 사용합니다. 
	- "기본값은 없지만, Field의 다른 설정들을 적용하고 싶어. 그리고 이 필드는 반드시 값이 있어야 해" 라는 뜻입니다. 
	- 만약 기본값을 주고 싶다면 `...` 대신 그 값을 넣어주면 됩니다. (예: `Field("기본 이름", min_length=3)`)

```
from fastapi import FastAPI
from pydantic import BaseModel, Field

app = FastAPI()


class Item(BaseModel):
    name: str = Field(
        ...,
        title="Item Name",
        min_length=3,
        max_length=50,
        example="게이밍 마우스"
    )
   
    description: str | None = Field(
        default=None,
        title="Item Description",
        max_length=300,
        example="이 마우스는 놀라운 성능을 자랑합니다."
    )
    price: float = Field(
        ...,
        gt=0,
        description="가격은 반드시 0보다 커야 합니다.",
        example=59.99
    )
    # 여기도 마찬가지입니다.
    tax: float | None = Field(
        default=None,
        ge=0,
        le=1.0,
        description="세금은 0과 1.0 사이의 값이어야 합니다.",
        example=0.1
    )


@app.post("/items/")
def create_item(item: Item):
    return item
```



- **`name: str = Field(...)`**
    
    - `...`: 이 필드는 필수입니다.
        
    - `title`: API 문서에 표시될 필드의 제목입니다.
        
    - `min_length=3`, `max_length=50`: 문자열의 최소/최대 길이를 제한합니다.
        
    - `example`: API 문서에 보여줄 예시 값을 지정합니다.
        
- **`description: ... = Field(default=None, ...)`**
    
    - `default=None`: `...` 대신 기본값을 `None`으로 지정하여 이 필드를 선택 항목으로 만들었습니다.
        
- **`price: float = Field(..., gt=0, ...)`**
    
    - `gt=0`: **g**reater **t**han 0. 즉, 0보다 커야 한다는 규칙입니다. `0`은 허용되지 않습니다.
        
    - `description`: 필드에 대한 상세 설명을 문서에 추가합니다.
        
- **`tax: ... = Field(default=None, ge=0, le=1.0)`**
    
    - `ge=0`: **g**reater than or **e**qual to 0. 즉, 0 이상이어야 합니다. `0`이 허용됩니다.
        
    - `le=1.0`: **l**ess than or **e**qual to 1.0. 즉, 1.0 이하이어야 합니다. `1.0`이 허용됩니다.


## 자주 사용하는 Field 인자 


| 인자 | 설명 | 사용 예시 |
| :--- | :--- | :--- |
| **첫 번째 인자** | 기본값 또는 필수 여부 지정 | `Field("기본값")` 또는 `Field(...)` |
| `title` | API 문서에 표시될 필드의 제목 | `title="상품명"` |
| `description`| API 문서에 표시될 상세 설명 | `description="가격은 0보다 커야 합니다."` |
| `example` | API 문서에 표시될 예시 값 | `example=99.9` |
| `min_length` | 문자열의 최소 길이 | `min_length=2` |
| `max_length` | 문자열의 최대 길이 | `max_length=100` |
| `gt` (`>`) | ~보다 크다 | `gt=0` |
| `ge` (`>=`)| ~보다 크거나 같다 | `ge=0` |
| `lt` (`<`) | ~보다 작다 | `lt=100` |
| `le` (`<=`)| ~보다 작거나 같다 | `le=100` |
